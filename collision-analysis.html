Bruno Simonサイトのコリジョン設定解析

Three.js + Cannon.js による効率的な物理演算の実装方法

🎯 調査結果の概要

Bruno Simonのポートフォリオサイトは、Three.jsとCannon.jsを使った優れた物理演算の実装例です。 特に注目すべきは、表示用モデルとコリジョン用モデルを完全に分離している点です。

重要な発見：Bruno Simonは複雑なConvexPolyhedronやTrimeshを使わず、 シンプルなBox、Cylinder、Sphereの組み合わせだけで正確なコリジョンを実現しています。

// Bruno Simonの車物理設定
const carBody = new CANNON.Body({
    mass: 40,
    shape: new CANNON.Box(new CANNON.Vec3(width/2, height/2, length/2)),
    position: new CANNON.Vec3(0, 0, 12)
});

// Z軸周りに90度回転
carBody.quaternion.setFromAxisAngle(new CANNON.Vec3(0, 0, 1), Math.PI / 2);

// RaycastVehicleを使用
const vehicle = new CANNON.RaycastVehicle({
    chassisBody: carBody
});

RaycastVehicleの特徴

  • 車体は1つのBoxコリジョン
  • ホイールは物理的な実体を持たない
  • レイキャストで地面との接触を検出
  • より軽量で安定した挙動

ホイール設定

  • 半径: 0.25
  • 高さ: 0.24
  • サスペンション剛性: 50
  • 摩擦スリップ: 10
  • タイプ: KINEMATIC

📦 命名規則による自動コリジョン生成

// Physics.jsの自動判定ロジック
let shape = null;

// メッシュ名から形状を自動判定
if(mesh.name.match(/^cube_?[0-9]{0,3}?|box[0-9]{0,3}?$/i)) {
    shape = 'box';
}
else if(mesh.name.match(/^cylinder_?[0-9]{0,3}?$/i)) {
    shape = 'cylinder';
}
else if(mesh.name.match(/^sphere_?[0-9]{0,3}?$/i)) {
    shape = 'sphere';
}

// 形状に応じた物理ボディを生成
if(shape === 'box') {
    const halfExtents = new CANNON.Vec3(
        mesh.scale.x * 0.5,
        mesh.scale.y * 0.5,
        mesh.scale.z * 0.5
    );
    shapeGeometry = new CANNON.Box(halfExtents);
}
注意：Bruno Simonは複雑な形状（ConvexPolyhedron、Trimesh）を使用していません。 全てのコリジョンはBox、Cylinder、Sphereの組み合わせで表現されています。

コリジョン用モデルの作成

各アセット（Tree.glb、Sign.glb等）に対して、シンプルな形状だけで構成された コリジョン用モデル（Tree_collision.glb等）を作成する

命名規則の採用

コリジョン用モデル内のメッシュに「box_01」「cylinder_01」等の名前を付ける

車のコリジョンをシンプル化

現在の方式（問題あり）

  • 1つのモデルから自動生成
  • バウンディングボックスの精度問題
  • ConvexPolyhedronの複雑さ
  • パフォーマンスの懸念

Bruno Simon方式（推奨）

  • 表示とコリジョンを分離
  • 手動で最適化されたコリジョン
  • シンプルな形状のみ使用
  • 軽量で高速
